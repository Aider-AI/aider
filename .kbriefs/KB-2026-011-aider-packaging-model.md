# Aider Packaging Model (Upstream Reference)

## Context & Problem Statement

aider-relay is a hard fork of aider. Before designing aider-relay's own distribution strategy, we need a precise picture of how aider itself is packaged and distributed. This knowledge is the baseline reference for deciding what to copy, adapt, or replace.

## Standard/Pattern Description

### Build System

aider uses **setuptools** with **setuptools_scm** for version management.

`pyproject.toml` (upstream repo `Aider-AI/aider`, path `/pyproject.toml`):

```toml
[project]
name = "aider-chat"
requires-python = ">=3.10,<3.15"
dynamic = ["dependencies", "optional-dependencies", "version"]

[project.scripts]
aider = "aider.main:main"

[tool.setuptools.dynamic]
dependencies = { file = "requirements.txt" }

[build-system]
requires = ["setuptools>=68", "setuptools_scm[toml]>=8"]
build-backend = "setuptools.build_meta"

[tool.setuptools_scm]
write_to = "aider/_version.py"
```

Key points:
- **PyPI name**: `aider-chat` (NOT `aider`). The `aider` name on PyPI is squatted/taken.
- **Entry point name**: `aider` — the CLI command users type.
- **Version source**: git tags via `setuptools_scm`. No hardcoded version. Running `pip install` from a commit without a tag produces a version like `0.85.1.dev42+gabcdef0`.
- **Dependencies**: loaded dynamically from `requirements.txt`, which is a pinned lockfile generated by `uv pip compile`.
- **No `setup.py`**: Pure `pyproject.toml`. setuptools is still the build backend (not Hatchling or Flit).

### Release Artifacts

Upstream releases (GitHub tag `v0.86.0`) produce **no binary assets** — the GitHub Release page has zero attached files. Distribution is entirely through PyPI (`pip install aider-chat`).

PyPI page: `https://pypi.org/project/aider-chat/` — ~4.9 million installs, current version 0.86.2.

### Installation Methods Supported by Upstream

Aider's official documentation at `https://aider.chat/docs/install.html` lists these methods in priority order:

**1. aider-install (recommended quickest path)**
```bash
python -m pip install aider-install
aider-install
```
`aider-install` is a separate PyPI package (`Aider-AI/aider-install`) that simply wraps `uv`:
- `install_requires = ["uv>=0.5.0"]`
- Entry point runs: `uv tool install --force --python python3.12 --with pip aider-chat@latest`

**2. One-liner scripts (recommended for new users)**
```powershell
# Windows
powershell -ExecutionPolicy ByPass -c "irm https://aider.chat/install.ps1 | iex"
```
```bash
# Mac/Linux
curl -LsSf https://aider.chat/install.sh | sh
```
These scripts install `uv` first (from `astral.sh/uv`), then run:
```bash
uv tool install --force --python python3.12 --with pip aider-chat@latest
```

**3. uv (direct)**
```bash
uv tool install --force --python python3.12 --with pip aider-chat@latest
```

**4. pipx**
```bash
pipx install aider-chat
```

**5. pip (with venv)**
```bash
python -m pip install -U --upgrade-strategy only-if-needed aider-chat
```

**6. System package managers (not recommended)**
brew, winget, choco, scoop — aider is available but often ships with incorrect dependencies.

### Docker Distribution

Aider's Dockerfile (`Aider-AI/aider/docker/Dockerfile`) uses:
- Base image: `python:3.12-slim-bookworm`
- Install method: `pip install /tmp/aider[help,browser,playwright]` (local copy baked in)
- **No Node.js in the Docker image** — upstream aider does not need Node.

### What aider-relay Must Diverge On

| Aspect | Upstream aider | aider-relay needs |
|--------|---------------|-------------------|
| PyPI name | `aider-chat` | `aider-relay` (new name) |
| Entry point | `aider` | `aider-relay` (or keep `aider` for drop-in compat) |
| Node.js | Not needed | Required (Claude Code + Codex are npm packages) |
| Python minimum | 3.10 | 3.12 (match aider's install.ps1 default) |
| Version source | git tags via setuptools_scm | Same approach viable |
| Build backend | setuptools | setuptools (no reason to change) |

## Rationale & Benefits

- Using the same build system (setuptools + setuptools_scm) as upstream minimises merge conflicts when pulling upstream patches.
- setuptools_scm versioning from git tags is the correct approach for a fork: version numbers can shadow upstream (`0.86.x-relay.1`) or be independent.
- The `aider-install` pattern (thin installer package that calls `uv`) is directly replicable for aider-relay.

## Anti-Patterns

### Incorrect: keeping `name = "aider-chat"` in pyproject.toml

If aider-relay is published to PyPI without renaming, it will conflict with upstream. Change `name` to `"aider-relay"` and the `[project.scripts]` entry point to `aider-relay = "aider.main:main"`.

### Incorrect: relying on system git for setuptools_scm in CI

setuptools_scm requires the full git history (or at least the most recent tag). In GitHub Actions, use `fetch-depth: 0` on the checkout step, otherwise version generation fails.

## Applicability Matrix

| Context | Applies | Rationale |
|---------|---------|-----------|
| Local dev (devcontainer) | No | Use `uv pip install -e .` directly |
| Windows end-user install | Yes | This is the reference for what to replicate |
